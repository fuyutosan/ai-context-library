# How to write the journal

The journal holds the raw record of what happened this month. Create one file per month named `journal/YYYY-MM.md`, and append an entry each time a piece of work wraps up.

## The format (hard limit: 5 lines)

```
## 2026-07-10 | Task name
- Did: what happened, in one line
- Output: file path or URL
- Learned: an insight worth reusing
- Shelf: work
```

- "Shelf:" is a note-to-self about where the entry gets filed during the weekly routine. Filling it in speeds things up later
- No raw logs (full conversations, full error messages). Essentials only
- Chat-only days and work not worth recording can be skipped

## Example entries

```
## 2026-07-12 | Built the library template
- Did: prepared the generic template for public release on GitHub
- Output: the template folder
- Learned: once personal details are stripped out, the README has to stand fully on its own
- Shelf: ai-tips

## 2026-07-15 | Finalized site A design
- Did: locked in the design draft with the client
- Output: `projects/site-a/design-final.pdf`
- Learned: asking for preferred colors upfront prevents rework
- Shelf: work
```

Save finished files right here in `journal/` as `YYYY-MM.md`. Never delete past months — keep them around for when you want to look back.
